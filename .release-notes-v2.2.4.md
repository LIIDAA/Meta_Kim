# v2.2.4 — Close v2.2.2 review evolution backlog (EB-001 + NEW-M1/L1/L2)

Documentation/contract-only patch closing 1 HIGH and 3 LOW findings from the v2.2.2 review evolution backlog.

## Why this patch

The v2.2.2 → v2.2.3 review chain identified an evolution backlog with one HIGH item (EB-001) and several LOW items (NEW-M1/L1/L2). EB-002/003/004 require spine-state.mjs changes and are explicitly deferred to v2.3.0. v2.2.4 closes the items that do NOT require production hook surface changes.

## Fixed

- **EB-001 — Worker per-file write-completion contract** — `config/contracts/workflow-contract.json` adds optional `fileCompletionListField` to `workerTaskPacket`, requiring workers with declared `scopeFiles` to report explicit per-file status (`completed` / `skipped` / `failed` + `skipReason`). `canonical/agents/meta-conductor.md` documents the narrative tier of this contract. Closes the v2.2.2 → NEW-H1 root cause where a worker silently dropped one of the planned target files (CHANGELOG was modified instead of `progress-v2.2.0.md`). Validator enforcement deferred to v2.3.0 R8.
- **NEW-M1 — CHANGELOG narrative accuracy** — The [2.2.2] section's release notes (`.release-notes-v2.2.2.md:71`) under-counted modified files at "9 files" by not including the 4 runtime mirror sync targets (`.claude/`, `.codex/`, `.cursor/`, `openclaw/`). Actual scope was ~13 files. Historical accuracy note added to the [2.2.2] section in CHANGELOG.md and CHANGELOG.zh-CN.md.
- **NEW-L2 — Release-notes consistency verifier** — `scripts/check-release-notes-consistency.mjs` (new) validates that `.release-notes-vX.Y.Z.md` files exist for each CHANGELOG section since v2.2.2 OR are documented as folded-into-README. Opt-in (`node scripts/check-release-notes-consistency.mjs`); not a mandatory CI gate until v2.2.5+.
- **NEW-L1** — Confirmed closed by v2.1.3 commit `bd70538f` (`Stable capability-index regeneration`). No residual gap; closure tracked here for completeness.
- **Historical fabrication acknowledgment** — Main-thread re-run of `npm run meta:check` revealed that `canonical/skills/meta-theory/SKILL.md:97` had been triggering a `validateNoHanOutsideAllowedTriggers` failure since v2.2.2. The v2.2.2 and v2.2.3 worker reports both claimed "20/20 pass" but never actually executed those commands. v2.2.4 closes the historical gap by extending `scripts/validate-project.mjs::isAllowedLocalizedTriggerLine` to allow the `方案 A` and `当前以聊天确认卡展示，不是弹窗` literals introduced in v2.2.2 for `tests/meta-theory/02-clarity-gate.test.mjs`. v2.2.4 is the first release where main-thread actually executed every claimed verification command. Worker dispatch contract gap captured as **EB-005** in v2.3.0 backlog.
